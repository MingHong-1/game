# 《星核守望》UI 与基础美工风格指南

> 阶段：2.5  
> 逻辑画布：1280 × 720，横屏 16:9

## 整体美术方向

核心关键词是“明亮星核幻想、雾林边境、清晰战斗信息”。环境以深蓝绿和雾青建立空间，交互结构使用半透明深蓝面板与柔和金色框线，能量、弹道和可操作状态使用明亮青蓝。画面可以有幻想装饰，但敌人、道路、星核生命和召唤状态必须优先可读。

禁止使用商业游戏素材、远程 CDN 图片、未授权字体和为了掩盖占位感而制作的大量一次性图片。当前雾林、道路、星核、单位与粒子均由 Phaser 程序图形生成。

## 设计令牌位置

| 职责 | 文件 |
| --- | --- |
| 颜色、语义色与提示色 | `src/ui/theme/uiTheme.ts` |
| 布局、间距、圆角、层级与动画时长 | `src/ui/theme/uiMetrics.ts` |
| 字体族与字号层级 | `src/ui/theme/uiTypography.ts` |

业务场景不得复制同一颜色或状态样式。需要新视觉语义时，先为主题增加有职责的令牌，再由组件消费。

## 颜色职责

| 语义 | 视觉方向 | 用途 |
| --- | --- | --- |
| 页面背景 | 深蓝绿、近黑蓝 | Canvas 底色、遮罩和阴影 |
| 战斗区背景 | 雾青、森林蓝绿 | 雾林环境与远景分层 |
| 普通面板 | 半透明深蓝 | 顶部栏、指挥台和普通弹窗 |
| 强调面板 | 稍亮蓝绿 | 重要状态与选中结构 |
| 面板边框 | 柔和金色 | 结构线和幻想科技边框 |
| 主操作 | 明亮青蓝 | 召唤、可操作状态和焦点 |
| 能量 | 青蓝 | 能量图标、数值增长和弹道光 |
| 星核生命 | 绿 → 黄 → 红 | 健康、警戒和危险生命阶段 |
| 星级/扩格 | 柔和金色 | 星级、奖励和新格位解锁 |
| 锁定/禁用 | 灰蓝 | 未解锁格和不可用按钮 |
| 危险 | 暗红/亮红 | 失败、Boss 警示和星核受伤 |

| 次要文字 | 浅灰蓝 | 标签、说明和战斗统计 |
| 提示文字 | 柔和金色 | 召唤原因、扩格说明和注意信息 |

## 字体与字号层级

只使用系统内置中文字体回退链：PingFang SC、Microsoft YaHei、Noto Sans CJK SC 和通用 sans-serif。不下载远程字体。

| 层级 | 建议字号 | 使用场景 |
| --- | ---: | --- |
| Caption | 11px | 图标标签、格位星级、调试信息 |
| Small | 13px | 次要状态、按钮小字、提示 |
| Body | 15px | 正文、战斗时间、波次信息 |
| Button | 16px | 常规按钮主文案 |
| Value | 18px | 资源数值和主操作 |
| Heading | 22px | 区域标题与重要提示 |
| Title | 36px | 弹窗和胜负标题 |
| Display | 72px | 主菜单游戏名称 |

## 面板规范

`Panel` 是所有结构容器的基础。默认使用半透明深蓝底、2px 边框、14px 圆角、轻微阴影和 14px 内边距。顶部状态栏与星核指挥台使用强面板；提示和普通弹窗使用正常面板；胜负面板通过 tone 改变边框，不复制面板实现。

面板只组织显示对象，不读取或修改战斗系统。面板高亮是展示状态，不能成为业务状态来源。

## 按钮状态

`GameButton` 统一处理以下状态：

- normal：深蓝底和结构边框。
- hover/focused：提高青蓝亮度并加粗焦点边框。
- pressed：轻微缩小并使用较深主操作色。
- selected：金色边框，供暂停和倍速选中状态使用。
- disabled：灰蓝化、降低透明度并关闭交互。
- danger：暗红底和红色边框。

按钮支持图标、主文案、次文案、键盘快捷键和 180ms 重复触发保护。禁用状态由 `BattleSession` 快照派生，按钮自身不决定是否可以召唤。

召唤按钮是最醒目的主操作：正常显示当前费用；能量不足和非扩格满格时禁用并显示原因；满格扩格边界保持可用，使用金色选中边框和“即将扩格”次文案。

## 英雄格位状态

`HeroSlotView` 使用统一 80 × 76 卡片，以容纳上下两排各五格：

- 空格：半透明深蓝底、青蓝细边框和淡星核纹路。
- 已占用：使用英雄主题色轻度高亮，几何英雄居中，名称与星级位置固定。
- 锁定：灰蓝底、半透明锁；只有下一个待解锁格显示剩余召唤次数。
- 新解锁：锁淡出、金色星光扩散，随后英雄淡入缩放出现。
- 选中：保留金色高亮接口，阶段 2C.2 不接入合成拖拽或站位属性。

十格从战斗开始时全部可见：下排第 1～5 格开放，上排第 6～10 格锁定并由左至右解锁。上下排共同以画布中心为中轴；资源与按钮对称布置在编队两侧。只有下一个待解锁格显示剩余召唤次数，更后续格只显示简化锁定状态。

局内星级只允许 1～4 星。4 星详情文案明确标记“满星”，不提供继续升星语义；满星只限制升星，不灰化格位，也不影响攻击、增益或未来遣散等非升星操作。星级显示统一消费 `HeroStarUiState`，不得在组件内维护独立的 1～5 数组或截断非法输入。

## 宽走廊规范

- 战场中心使用顶部较宽、中段稳定、底部适度收束的连续面状走廊。
- 走廊边缘清晰，内部通道只用低透明星轨暗示，不绘制成五条硬分隔公路。
- 默认 5 条隐形通道由同一中心路径派生；3 条和 7 条布局继续复用同一绘制方式。
- 普通敌人满血时隐藏小生命条，受伤后显示；重甲、精英和 Boss 始终保留简洁生命信息。
- 两侧雾林、岩石与雾气继续使用程序图形，正式背景接入后可替换环境层而不改变通道几何。

格位只显示 `BattleSessionSnapshot.slots`，不能直接操作英雄袋、能量或战斗英雄。

## 资源显示规范

`ResourceDisplay` 由图标、语义标签、快照数值和复用的差值文字组成。能量和费用直接读取 `BattleSession`，组件不维护第二份业务余额。数值变化可以放大和显示 +1/-5，但动画结束值不参与资源结算。

资源不足时只改变边框与提示色，不修改按钮业务状态。未来金币、星尘等资源继续复用该组件。

## 生命条与提示

`HealthBar` 接收真实 current/maximum，并平滑调整显示比例；胜负和受伤判定仍由战斗模拟完成。星核生命在顶部栏集中展示，战斗区星核本体根据健康比例使用绿、黄、红状态。

`BattleNotice` 使用上限为 4 的有界队列。同文案会复用最新位置，提示顺序显示，不允许无限堆积。用于能量/召唤反馈、扩格、波次、Boss 和战斗警示。

## 动画时长建议

| 类型 | 时长 |
| --- | ---: |
| 按钮按压、敌人受击 | 120ms |
| 常规数值、面板、锁淡出 | 220ms |
| 数值浮动、星光扩散 | 420ms |
| 普通提示停留 | 1050ms |
| 波次/Boss 提示 | 1450ms |

循环动画只允许用于低强度雾气和星核呼吸光。禁止长期强闪、长时间镜头震动和阻塞玩家操作的过场。

## 临时视觉与长期框架

当前临时视觉：

- 四名英雄的彩色几何图形。
- 敌人和 Boss 的几何轮廓。
- 程序绘制雾林、岩石、道路纹理和星核结构。
- 无正式音效和最终粒子。

可长期保留：

- 主题令牌与语义颜色。
- `Panel`、`GameButton`、`ResourceDisplay`、`HeroSlotView`、`HealthBar`、`BattleNotice` 组件契约。
- 快照 → UI 派生状态 → 组件更新的数据流。
- 有界提示队列、动画清理和按钮重复触发保护。

## 正式资源接入方式

阶段 2.5 已建立视觉定义与显式 Manifest；阶段 2.5B 将视觉节点冻结为 1/3/4 星。`HeroSlotView` 内部由 `HeroBattleView` 按星级选择正式资源：1～2 星使用 1 星图，3 星按 3→1 回退，4 星按 4→3→1 回退，最后使用程序图形。图片只替换英雄视觉内容，名称、星级、边框、锁定、选中和格位坐标仍由 UI 统一绘制。

`EnemyBattleView` 保持运动根节点与视觉容器分离。图片、Sprite Sheet、受击、缩放、闪白和死亡淡出只作用 `VisualContainer`；`EnemyRootContainer` 的世界位置仍只由移动插值系统写入。主题背景位于道路、敌人、星核和 UI 下方，不能烘焙或改变权威路径坐标。

所有不同来源的英雄/怪物图片必须通过视觉定义配置缩放、脚底锚点、格位偏移、生命条偏移和最大体型，不允许在 `PrototypeScene` 为单个资源写临时坐标分支。头像不烘焙 UI 边框、文字或星级；音频无资源时安全静默。完整资源尺寸、命名、许可和启用流程见 `docs/asset-pipeline.md`。

正式资源只影响表现，不改变职业攻击区域、召唤结果、随机流、伤害、移动或战斗状态。纹理、动画或音频加载成功与否不能决定实体是否存在。
